# Найденные баги:
- 1: item_discount вообще не учитывается при подсчете цены

- 2: При изменении exchange_rate происходит неправильная конвертация валюты (округляется всегда в меньшую сторону)

- 3: Регистр vend_cfg имеет неправильное значение после сброса (поле idle_timeout имеет значение старой версии, на работу не влияет)

- 4: Чтение регистров vend_item выдает неправильную информацию

- 5: Размерность поля exchange_rate в DUT не соответсвует документации (на работу не влияет)

- 6: Поле item_count не учитывается при работе автомата (сигнал item_empty не поднимается, когда товар закончился)

- 7: Сдача не возвращается, если истёк таймаут на ожидание (автомат продолжает работать)

# Вопросы по спецификации:
- Формат округления сдачи
- Поведение автомата после установки сигнала item_empty (вернет ли деньги?)
- Поведение автомата после истекания таймаута на ожидание (в какой момент возвращаются деньги?)
- Что будет при внесении монеты недоступного номинала
- Что происходит при попытке купить товар, если средств недостаточно (на данный момент автомат перестает отвечать)
